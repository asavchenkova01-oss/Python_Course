What is Git and what problem does it solve?
Git is VCS - system for developers to control changes made to their code. It helps solve code overwrite problems.

What is a version control system (VCS)?
A version control system is a software tool that records changes to files over time so that you can recall specific versions later. It includes history of changes made to a project enabling i.e. developers to track code overwrites, modifications.

What is a repository (repo)?
A repository is like a digital, cloud folder which stores all the modified versions of your uploaded project. With keeping repository public and sharing the link with others we can easily share the project with others.

What is a commit? Why do we write a commit message?
We use commit to upload changes, latest updates of our project. We use commit message to add specific comment on what exactly was changed, modified.

What is the difference between git add and git commit?
git add chooses and prepares specific modified files that you want to include in your next save. git commit takes those prepared files and officially saves them in your project's history.

What does git push do? What does git pull do?
git push sends your locally saved commits up to a remote platform like GitHub so others can see them. git pull does the opposite—it grabs the newest updates from GitHub and brings them down to your local computer to update your code.

What is a branch and why is it useful?
A branch is like a parallel copy of your project where you can work on a new feature safely. It is useful because you can experiment and write code without breaking the main, working version of your project.

What is GitHub?
What is GitHub and how is it different from Git?
GitHub is an online website used to store, host, and share your Git repositories in the cloud. The difference is that Git is the actual tool running locally on your computer to track changes, while GitHub is the online platform where you upload those changes.

Name two alternatives to GitHub (e.g. other websites that host Git repos).
Two other popular websites that host Git repositories are GitLab and Bitbucket.

What is the difference between a public and a private repository?
A public repository can be seen, downloaded, and read by anyone on the internet. A private repository is hidden, meaning only you and the specific people you invite can see or access the code.

What is a README.md file and what is it used for?
A README.md file is the introduction text page of your repository. It is used to describe what your project is about, what features it has, and how other people can install or run it.

What is a .gitignore file and why would you not want to commit every file?
A .gitignore file is a list that tells Git which files to completely ignore so they don't get uploaded. You don't want to commit everything because some files contain secret passwords/API keys, and others are heavy.
